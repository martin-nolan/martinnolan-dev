# Overview

This harness is a portable set of guidance for agent-assisted engineering work.

It is meant to be copied into a repository and adapted to that repository's real structure, commands, and review expectations. It does not assume a specific application stack.

## What It Provides

- repo-local read-order guidance
- reusable workflow skills
- review-readiness expectations
- workstream and requirement templates
- scratchpad patterns for temporary task notes
- documentation hygiene rules
- lightweight harness evaluation prompts

## What It Does Not Provide

- application architecture
- deployment infrastructure
- product-specific conventions
- generated code
- guaranteed command surfaces

Those belong in the target repository after the harness is adapted.

## Adaptation Rule

Before using this harness in another repo, replace generic placeholders with that repo's actual:

- module names
- commands
- test expectations
- documentation ownership
- release or deployment checks
- active assistant/tooling conventions
