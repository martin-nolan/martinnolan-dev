# Review Readiness

Use this checklist before handing off an agent-assisted change.

## Required Evidence

- Scope is clear.
- The touched files match the stated scope.
- The change has been verified with relevant commands.
- Any skipped check has a concrete reason.
- Docs match the behavior that now exists.
- Risks and follow-ups are named.

## Handoff Format

Keep the final handoff short:

```md
Changed:
- ...

Verified:
- ...

Notes:
- ...
```

## Blocked Handoff

If the change is blocked, state:

- what was attempted
- the exact failure or missing input
- what is needed next
- whether any partial changes should be kept or reverted

Do not hide uncertainty behind polished wording.
